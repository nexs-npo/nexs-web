## 1. Product Identity (プロダクトの定義)

### What is this?
**「法人のデジタルツイン（Digital Twin of the Organization）」**

nexs-webは、単なるNPO法人の公式サイトやオウンドメディアではありません。
物理的な制約を持たない「nexs」という組織そのものを、デジタル空間上に構築・実装するプロダクトです。

意思決定、資金の流れ、研究プロセス、知の蓄積。
組織のあらゆる活動がこのWebアプリ上で展開され、ユーザーはそれをリアルタイムに体験・監視・参加します。

### Core Philosophy
**"Management without Management" (管理なき管理)**

* **Transparency as Governance:** すべてを公開し、衆人環視（Public Audit）の状態を作ることで、管理コストをゼロにしつつ、最高の統治を実現する。
* **Open by Default:** プロセス、失敗、議論、そして会計。隠すコストよりも、公開して信頼を得るメリットを選択する。

## 2. UX Concept (ユーザー体験のコンセプト)

### Metaphor: "The Glass Laboratory" (ガラス張りの研究所)
ユーザーがこのアプリを開くことは、nexsという**「透明な壁でできた研究所」に入館すること**と同義です。

来訪者は、エントランスでnexsの現在（/index）に触れ、
ガラス越しに研究員たちが実験している様子（/lab）を眺め、
書庫（/library）で成果物を読み耽り、
時には執務室（/office）の金庫の中身や、決裁の瞬間さえも目撃します。

### User Journey
1. **Visit (入館):** nexsアプリを開き、組織の「いま」を確認する（nexs タブ）。
2. **Explore (回遊):** 関連する研究プロジェクトや、組織の裏側（予算・議案）へシームレスに移動する。
3. **Trust (信頼):** 嘘のない運営実態を見て、組織への信頼を深める。
4. **Join (参加):** 自分の席（My Desk）を持ち、寄付やプロジェクトへの参加を行う。

アプリのナビゲーションは、研究所の「空間」を模して設計されています。

## 4. Governance Mechanics (ガバナンスの仕組み)

このプロダクトの最大の特徴は、ガバナンス機能がバックオフィス（管理画面）ではなく、フロントエンド（ユーザー体験）として提供される点です。

* **Governance as Code:**
    * 法人の意思決定は「会議室」ではなく「Gitリポジトリ」で行われる。
    * 議案の可決（Merge）プロセスはすべてコミットログとして可視化される。
* **Real-time Dashboard (将来実装):**
    * 財務状況はAPI経由でリアルタイムにダッシュボード化される。
    * ユーザーは「支援したお金が今どう使われているか」をいつでも確認できる。
* **Zero Trust Architecture:**
    * 人間もAIも信用しない前提で設計する。
    * 静的サイト生成（Astro）を基本とし、動的な脆弱性を排除する。
    * 個人情報（PII）は保持せず、外部認証基盤（Clerk）に委任する。

## 5. Technical Principles (技術指針)

* **Mobile First:** 複雑なガバナンス情報も、スマホで直感的に閲覧できるUIとする。
* **Static & Fast:** AstroによるSSG（静的サイト生成）をベースとし、圧倒的な表示速度とセキュリティを担保する。
* **Git-based Truth:** 重要なデータ（議案・記事）の正はGit上のファイル（Markdown/MDX）とする。一部の動的データはSupabase（PostgreSQL）で管理する。

## 6. Success Metrics (成功定義)

このプロダクトが「組織のデジタルツイン」として成功したと言える状態：

### 1. Transparency (透明性の実現)
- 組織の全ての意思決定、予算配分、プロジェクト進捗が、誰でも・いつでもアクセス可能な状態になっている
- 「隠された情報」が存在しないことが、外部からも確認できる
- ユーザーが疑念を持たずに組織を信頼できている

### 2. Ubiquity (遍在性・分散性)
- メンバーが物理的に集まらなくても、組織の「現在」をリアルタイムに把握できている
- 時差・場所・デバイスを問わず、同じ情報・同じ権限でアクセスできる
- 「いまnexsで何が起きているか」を、いつでもこのアプリで確認できる

### 3. Autonomy (自律性)
- メンバーが上長の承認を待たずに、公開されたルールと予算内で自律的にプロジェクトを推進できている
- 意思決定プロセスが透明化され、「誰かに聞かないと分からない」状態がゼロになっている

### 4. Trust Conversion (信頼の転換)
- 訪問者が透明な運営実態に触れて、「寄付者」「協働者」に転換している
- 「見える化」が新しい支援者の獲得につながっている
